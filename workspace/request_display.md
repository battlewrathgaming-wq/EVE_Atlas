Aura Lab accepts advisory display requests through:

`F:\Projects\AURA- Lab\workspace\request_display.md`

Use `request_display` only when Atlas wants Lab to compare Bridge -> Interface presentation options for an Atlas-owned current presentation surface or use case.

Authority:

- Atlas owns internal -> Bridge meaning, source terms, Evidence, Discovery, Watch, Marked, provenance, storage, and renderer semantics.
- Lab may suggest Bridge -> Interface display methods only after Atlas meaning is preserved.
- A request is advisory input, not implementation approval, adoption approval, bridge contract, payload schema, or terminology rename.

Workflow:

1. Create or propose one `request_display` entry using the Lab schema.
2. Preserve Atlas-owned terms and list any terms Lab must not rewrite.
3. Send the entry to Lab for use-case comparison.
4. Lab maps the request to Lab slots, display types, material sets, and up to three candidate display methods.
5. Atlas reviews any recommendation under Atlas authority before implementation.

Do not treat this pointer as an active Atlas task queue.

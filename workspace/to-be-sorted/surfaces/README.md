# Atlas Surface Discovery Map

Status: Advisory discovery input, not project authority

This folder maps Atlas surfaces that are user-facing now or likely to become user-facing later. It is intended as lightweight input for future UIUX, Lab, or display-request work.

This folder does not authorize Dev work, implementation, renderer redesign, bridge changes, command changes, source-term renames, Lab adoption, or final UI specification.

## How To Read

- Start with `atlas-surface-index.md`.
- Use each surface note as a first-pass interpretation of what an operator needs to understand.
- Preserve Atlas source meanings even when later presentation language changes.
- Treat request_display notes as candidates only. No request has been submitted from this folder.

## Boundaries Preserved

- Discovery refs are zKill possible leads and provenance.
- Evidence/EVEidence is ESI-expanded killmail truth written locally.
- Hydration repairs readability over known local IDs.
- Observation is story/query over connected records, not new truth.
- Assessment Memory is human-authored judgment, not proof.
- External I/O is provider contact authority.
- Storage setup is user trust/storage authority, not only a path variable.

# Term: Collection Provenance

## Plain Meaning

Collection provenance explains how evidence entered AURA Atlas.

It answers:

> How did Atlas find and collect this evidence?

## Examples

Collection provenance can include:

- run ID
- collection time
- zKill route used
- systems scanned
- API calls made
- warnings
- caps or skipped expansions

## Why It Matters

Provenance helps audit collection behavior and explain partial samples.

But provenance is not the same as intelligence scope.

## Simple Difference

Collection provenance:

> This killmail was found during a radius watch run.

Evidence scope:

> This killmail happened in ZTS-4D during this time window.

## Product Rule

Run reports explain collection provenance.

Evidence and observation reports should be scoped by what the evidence describes: system, radius, actor, region, and time.

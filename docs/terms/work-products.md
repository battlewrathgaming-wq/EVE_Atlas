# Term: Work Products

## Plain Meaning

A work product is something AURA Atlas produces or preserves.

Not all work products mean the same thing.

AURA Atlas uses three layers:

```txt
Evidence
-> Observation
-> Assessment
```

The product as a whole is an intelligence tool, but its individual outputs should use precise names.

## Evidence Work Products

Evidence work products answer:

> What stored evidence do we have?

Examples:

- expanded killmails
- activity events
- evidence reports
- run reports
- collection provenance
- ingestion audits

Good wording:

- evidence
- stored evidence
- evidence scope
- evidence report
- activity event
- partial sample

Evidence work products should not contain conclusions beyond the evidence.

## Observation Work Products

Observation work products answer:

> What patterns are visible in the evidence?

Examples:

- observed operators
- repeated appearances
- repeated corporations
- multi-system presence
- attacker/victim role mix
- activity timelines

Good wording:

- observed
- repeated presence
- candidate signal
- pattern
- role mix
- observation report

Observation work products show patterns. They do not prove intent by themselves.

## Assessment Work Products

Assessment work products answer:

> What do we think this means, and what should we remember or act on?

Examples:

- entity interest
- priority score
- impact score
- watchlist promotion
- analyst note
- hot/warm/cold trail
- confidence-bearing recommendation

Good wording:

- assessment
- interest
- priority
- impact
- confidence
- interpretation
- recommendation
- assessment artifact

Assessment work products should be deliberate and explainable.

They should not be silently generated for every observed entity.

## Why This Matters

Calling everything an intelligence report can make the system sound more certain than it is.

AURA Atlas should be precise:

- evidence reports show evidence
- observation reports show patterns
- assessment artifacts preserve judgment

This helps the user understand what is known, what is observed, and what is being inferred.

## Product Rule

Use the lowest, most precise layer that fits.

If Atlas only has evidence, call it evidence.

If Atlas sees a pattern, call it an observation.

If Atlas or the user assigns meaning, priority, or interest, call it assessment.

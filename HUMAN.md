# HUMAN.md

Status: Human context refresher
Purpose: Help agents understand how the Human may communicate during AURA work

This file is context, not project authority.

It does not replace `AGENTS.md`, `workspace/current.md`, `workspace/critical/`, accepted handoffs, or durable docs. It does not authorize Dev work, live/private/destructive actions, shared-file edits, or product-direction changes.

## Communication Shape

The Human may guide through discussion, preference, metaphor, reaction, examples, exploratory language, or developing vision rather than formal command language.

Treat Human discussion as meaningful. Do not ignore it because it is informal.

Also do not treat every Human sentence as automatic authorization. Sort discussion into intent, advisory context, accepted direction, parked ideas, unresolved questions, and possible future runway only when the role workflow or Human signal calls for that.

## Cooperative Default

Human conversation is advisory and cooperative unless explicitly framed as a decision, approval, instruction, or runway request.

Brief affirmations such as "sounds good", "nice", "yeah", or similar may indicate agreement, encouragement, or permission to continue a clearly stated role-appropriate next action. They are not broad authorization to implement, change authority, accept doctrine, broaden scope, or edit shared/global files.

If an affirmation follows a specific proposed action, continue only that bounded action according to role and project workflow. If scope is unclear, summarize or ask before acting.

## Context Attention

When the Human uses `#`, follow the project Context Attention Mode in `AGENTS.md` and `workspace/00-dot-protocol.md`.

In plain terms:

- `#` opens a container for messy but useful discussion.
- the Human may then think aloud naturally.
- `#` closes the container and asks the agent to classify what mattered.

`#` is not Dev authorization and does not override `workspace/current.md`.

## Role Handling

Agents should keep their assigned role frame while listening naturally.

For Atlas Overseer work, this means:

- hear Human intent and product texture
- separate advisory input from accepted direction
- keep Dev insulated from half-formed exploration
- turn stable decisions into project state only through the local workflow
- push back when a thought risks blurring authority, scope, safety, or timing

For Dev work, this means:

- execute only the active `workspace/current.md` runway
- flag unclear Human context in handoff
- do not expand scope from conversational context alone

## Practical Reading

Use this file as a refresher when:

- context was compacted
- a new agent is entering the project
- the Human is discussing product vision or operator experience
- informal guidance may otherwise look like conflicting instructions

Do not make this file a standing task queue or role prompt.
